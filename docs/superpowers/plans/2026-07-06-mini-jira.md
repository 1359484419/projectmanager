# 小型自托管 Jira（projectmanager）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 ~10 人小团队构建多租户自托管 Jira 替代品：Sprint/point 容量规划、Epic/季度路线图、四态工作流、Dashboard/All Sprints、内置 MCP 接口 + Agent Skill。

**Architecture:** 单体 Spring Boot 3（Java 21）+ React SPA + PostgreSQL 16。多租户为共享库 + `tenant_id` 列，TenantContext 拦截器 harness 统一注入过滤；MCP Server 内置于同一应用（Streamable HTTP `/mcp`），PAT 认证复用同一 harness。每日调度 job 负责 Sprint 自动轮转（幂等）。

**Tech Stack:** Java 21, Spring Boot 3.3（Web/Security/Data JPA）, Flyway, PostgreSQL 16, jjwt 0.12, MCP Java SDK（Spring WebMVC transport）, React 18 + Vite + TypeScript, TanStack Query v5, dnd-kit, Recharts, Testcontainers, Playwright。

**Spec:** `docs/superpowers/specs/2026-07-06-mini-jira-design.md`（本计划的需求真源，冲突时以 spec 为准）

## Global Constraints

- 所有业务表带 `tenant_id NOT NULL REFERENCES tenants(id)`；业务代码不得手写 tenant 过滤，必须走 harness（Hibernate `@Filter` + TenantContext）。
- 跨租户访问一律返回 **404**（不是 403）。
- 1 point = 1 人天，正整数。容量提示性，不硬阻止超配。
- 任务四态：`TODO → IN_PROGRESS → COMPLETED → DONE`，允许回退；燃尽图/Epic 进度/Sprint 结转只认 DONE；进入 DONE 写 `done_at`，回退清空。
- Sprint：同项目最多一个 ACTIVE；项目默认周期 `WEEK_2`；`auto_rotate` 默认 true；轮转 job 必须幂等（停机补跑只轮转一次到当前周期）。
- 所有任务变更（状态/points/sprint/指派/创建）写 `activities`；MCP 来源要标记。
- API 前缀：认证 `/api/auth/*`、个人 `/api/me/*`、租户内 `/api/t/{slug}/*`；统一错误体 `{code, message}`。
- `create_tasks`（MCP）单次批量上限 20 条。
- 提交信息用中文或英文均可，遵循 conventional commits（feat/fix/docs/test/chore）。
- 每个任务完成即 commit；后端测试用 Testcontainers 跑真实 PG（本机需 Docker）。

### Task 4: TenantContext 拦截器 harness（核心）

**Files:**
- Create: `pm/tenant/TenantContext.java`, `TenantInterceptor.java`, `TenantEntity.java`, `TenantFilterAspect.java`, `pm/common/ApiException.java`, `GlobalExceptionHandler.java`
- Test: `pm/tenant/TenantIsolationTest.java`

**Interfaces:**
- Produces（后续所有租户内功能都建立在这上面）:
  - `TenantContext.require() → long tenantId`（ThreadLocal，请求结束清理）
  - `@MappedSuperclass TenantEntity { @Column(name="tenant_id") Long tenantId; }`，实体上加 Hibernate `@Filter(name="tenantFilter", condition="tenant_id = :tenantId")`
  - `TenantInterceptor`：匹配 `/api/t/{slug}/**` → 查 tenant → 校验当前用户 membership（无则抛 `ApiException.notFound()`）→ `TenantContext.set(...)` → 开启 Hibernate session filter；`afterCompletion` 清理。
  - `TenantFilterAspect`（或 `OpenEntityManagerInViewFilter` 后置）：确保每个事务 session 都 `enableFilter("tenantFilter").setParameter("tenantId", ...)`。实现方式：`@Component` 实现 `HandlerInterceptor` + 一个 `TransactionSynchronization`/`SessionFactoryObserver` 均可，最简做法是在 Service 层入口用 AOP 对 `pm..*Service.*` 开 filter——选定：**在 `TenantInterceptor.preHandle` 里通过注入的 `EntityManager` unwrap `Session` 开 filter，配合 OSIV（默认开启）**，一处生效全请求。
  - 保存时自动填 tenant：`@PrePersist` 在 `TenantEntity` 里 `if (tenantId == null) tenantId = TenantContext.require();`
  - `GlobalExceptionHandler`：`ApiException → {code, message}`；未知异常 → 500 `{code:"INTERNAL"}`。
- Consumes: Task 3 的 JWT 认证与 memberships。

- [ ] **Step 1:** 写失败测试 `TenantIsolationTest`：建租户 A（用户 a）与租户 B（用户 b）；用 b 的 token 请求 `GET /api/t/{aSlug}/ping` → **404**；a 请求自己 → 200。（`/api/t/{slug}/ping` 是本任务加的最小租户内测试端点，返回 `{tenantId}`。）

### Task 10: Sprint 自动轮转 job（幂等）

**Files:**
- Create: `pm/sprint/SprintRotationJob.java`, `pm/config/SchedulingConfig.java`

**Interfaces:**

- Consumes: Task 9 `SprintService.rotate`。
