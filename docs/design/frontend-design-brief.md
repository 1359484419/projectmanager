# PM（mini-Jira）前端设计 Brief

> 交给视觉/UI 设计的输入文档。功能与 API 已实现并部署，本次只重新设计 UI 皮肤与交互细节，**不改功能范围、不改路由结构、不改 API**。

## 1. 产品定位

多租户敏捷项目管理工具（自托管 Jira 替代品），用户是 ~10 人的软件团队，每天高频使用：看进度、拖看板、规划 Sprint。密集使用的生产力工具，不是营销页。

**设计方向**：现代 SaaS 生产力工具审美，对标 **Linear / Height / Plane**——克制、紧凑、信息密度高、大量留白靠层次而非装饰。明确不要传统企业后台（AntD 蓝色官僚风）的感觉。

**技术承接约束**（设计产出需可落地到这套栈）：

- shadcn/ui + Tailwind CSS（已选型），Radix 交互原语
- 图表 Recharts、拖拽 dnd-kit（保留，不换库）
- 桌面优先（≥1280px 为主场景），平板可用即可，不做手机端专门设计
- **需要暗色模式**（dark 为默认更符合目标用户，light 也要完整）
- 图标建议 lucide

## 2. 全局结构

```
┌──────────┬──────────────────────────────────────┐
│          │ 顶栏：当前项目切换 ▾ · 全局搜索(预留) · 用户菜单 │
│  侧边栏   ├──────────────────────────────────────┤
│  (fixed) │                                      │
│          │              页面内容区                │
└──────────┴──────────────────────────────────────┘
```

- **侧边栏**（约 220px，可考虑折叠态 56px）：租户名/Logo 区、导航（Dashboard、Backlog、看板 Board、All Sprints、规划 Planning、报表 Reports、路线图 Roadmap，分隔线，管理 Admin、设置 Settings）、底部当前用户。当前项高亮。
- 路由前缀 `/t/{tenantSlug}/...`，顶栏含项目切换下拉（多数租户只有 1 个项目，切换器要低调）。
- 认证页（登录/注册/接受邀请）是独立无侧边栏布局：居中卡片，一个品牌名 "PM"（可给个简洁字标）。

## 3. 全局语义色 & 记号系统（贯穿所有页面，必须统一）

**任务状态四态**（最重要的视觉语言）：

| 状态 | 语义 | 建议方向 |
|---|---|---|
| TODO | 未开始 | 中性灰 |
| IN_PROGRESS | 进行中 | 蓝 |
| COMPLETED | 开发完待验收 | 琥珀/紫（要与 DONE 明显区分） |
| DONE | 验收通过 | 绿 |

**任务类型**：STORY（绿色书签/故事图标）、BUG（红色虫）、TASK（蓝灰勾选框）——Jira 用户的肌肉记忆，沿用这个色彩直觉。

**其他语义**：容量超载 = 红 + 斜纹；Sprint 状态徽标 PLANNED（灰）/ ACTIVE（蓝，带点脉冲感）/ CLOSED（中性淡）；Epic 有用户自选色条（6 色调色板，请给一套和谐的）。

**任务卡（TaskCard）是全站复用原子**，出现在看板/Backlog/Dashboard/规划/路线图：类型图标 + 编号（`PM-42`，等宽字体，次要色）+ 标题 + points 徽标（小圆片）+ 负责人头像（首字母，无头像图）。请给紧凑（列表行）与卡片（看板）两种形态。

## 4. 页面清单与要点

### 4.1 Dashboard（项目首页）
- 顶部统计条：当前 Sprint 名称、剩余天数（醒目）、四状态计数、整体完成度（donePct 环形或进度条）。
- 主体：Story/Bug 按四状态分为四组的只读列表。
- 空态：无 ACTIVE Sprint → 引导去创建（空态插画风格：简洁线条，不要卡通）。

### 4.2 Backlog
- 列表页：顶部快速创建行（类型下拉 + 标题输入 + points，回车即建——参考 Linear 的 inline create）、类型筛选胶囊、任务行（紧凑 TaskCard + 「移入 Sprint」操作）。
- 信息密度优先：一屏尽量多行，行高 ~36-40px。

### 4.3 Board（看板，核心页面）
- 四列拖拽（TODO/IN_PROGRESS/COMPLETED/DONE），列头 = 状态色点 + 名称 + 计数。
- 拖拽反馈：拿起卡片阴影抬升、目标列高亮、空列虚线占位。
- 这是用户停留最久的页面，请重点打磨。

### 4.4 All Sprints
- Sprint 倒序分组卡：状态徽标 + 名称 + 起止日期 + 任务数，默认全展开可折叠；组内任务表格（编号/类型/标题/状态/points/负责人）。
- 含「新建 Sprint」与 PLANNED 卡上的「启动」按钮。

### 4.5 Planning（规划）
- 双栏：左 Backlog 简表（可拖），右当前/下个 Sprint 容器（可放）。
- **成员容量条**（本页灵魂）：每成员一行，头像 + 名字 + 条形 `已分配/容量`，≤100% 填充色，>100% 红 + 斜纹超出段；点数字行内编辑 override。请给这个组件精细的规范。

### 4.6 Reports（报表）
- Sprint 选择器 + 燃尽图（remaining 实线 + ideal 虚线，网格极淡）+ 每人负载横条图。
- 图表配色需与语义色系统一致，暗色模式下也要可读。

### 4.7 Roadmap（路线图）
- 按季度分组（"2026-Q3"）的 Epic 卡：左色条 + 名称 + 进度条（donePoints/totalPoints）+ 展开任务列表；「未指定季度」组置底。
- 新建 Epic 模态：名称/描述/季度/颜色选择。

### 4.8 TaskDrawer（任务详情抽屉，全站复用）
- 右侧滑出（~480px）：标题行内编辑、属性栏（状态/类型/points/负责人/Epic/Sprint，全部行内下拉）、描述、Tab（评论 | 变更历史）。
- 变更历史渲染成时间线：「张三 把状态从 进行中 改为 已完成 · 3 小时前 · via MCP」（MCP 来源给个小标记）。

### 4.9 Admin（租户管理）与 Settings（个人设置）
- 常规设置页型：分区卡片。Admin：成员表、邀请链接生成（生成后显示可复制链接）、项目设置（默认周期/自动轮转开关）、新建项目。
- Settings：改名/改密码、**PAT 管理**——生成后明文只显示一次的 token 展示样式（等宽、高亮、复制按钮、"只显示一次"警示）请专门设计。

### 4.10 认证页
- 登录/注册双 tab（注册含团队名 + slug）、接受邀请页。居中卡片，干净。

## 5. 通用状态规范（每页都要）

- Loading：skeleton（不要 spinner 满屏）
- Error：内联错误条 + 重试
- 空态：一句话 + 主操作按钮，插画可选
- Toast：操作成功/失败反馈（目前缺失，请纳入设计）
- 破坏性操作（吊销 PAT、关闭 Sprint）：确认对话框

## 6. 交付物期望

1. 设计 token：色板（含四态/类型语义色、Epic 6 色）、字号阶梯、间距、圆角、阴影，light + dark 两套
2. 核心组件规范：TaskCard 两形态、StatusBadge、TypeIcon、CapacityBar、Sprint 卡、TaskDrawer、快速创建行
3. 关键页面高保真：Board、Dashboard、Planning、Backlog（其余页面可用组件规范推导）
4. 产出若为 HTML/React mock 最佳（便于直接翻译成 shadcn/Tailwind 实现）
